## 1. 개요
REES46 화장품 쇼핑몰 행동로그(view·cart·remove·purchase)를 **이탈(churn)·카테고리/아이템 추천·세션 바운스** 과제용 학습데이터로 가공했다. 모든 전처리는 본 프로젝트에서 수행하고, 각 모델군에 대해 **베이지안 전처리·하이퍼파라미터 탐색(per-option ablation 포함)** 을 돌려 최적 조합과 그 결과를 기록했다. 산출물은 parquet/npz + meta/bayes JSON으로 학습팀에 인계한다.

## 6. 과제별 상세

### 6.1 7일 이탈(churn) — v4_model_prep / v2_5m
- **라벨**: churn = 관측 종료 후 **7일 무활동**(recency 기반).
- **분할(시간외삽)**: train 2019-10-01~2020-01-25(+out7d), test **2020-02**(OOT). 미래 누수 차단.
- **규모**: train **1,266,514** users(churn **96.67%**), test **298,813**(churn **90.21%**).
- **활성 코호트(recency≤7)**: n_train **109,378** / n_test **109,736**, churn **82.17%** — 전체 이탈률이 과도(96.7%)해 "최근 활동했지만 이탈" 코호트로 평가 의미 확보(§8).
- **피처**: 정본 **22**(§7), 5m 요약 **10**.

### 6.2 카테고리 추천 — v4-1
- **타깃**: y_next_category = outcome 구간 최빈 category_id. **389 클래스**(MIN_COUNT≥10 필터 후).
- **규모**: train **42,186** / test **29,205** rows. **outcome 커버리지**(active 유저 중 라벨 보유) train **3.33%** / test **9.77%**.
- **불균형**: 최빈 클래스 2,201건, majority top-1 baseline **5.54%**.

### 6.3 아이템 추천 — v4-2
- **타깃**: y_next_item = 다음 최빈 product_id, **train 상위 1000개 상품**. **1000 클래스**.
- **규모**: train **20,254** / test **11,159** rows. **top-K 커버리지** train **48.0%** / test **38.2%**(상위 1000에 정답이 포함될 확률). majority top-1 **4.68%**.

### 6.4 세션 바운스(churn30) — v4-3
- **라벨**: churn30 = 이 이벤트 직후 **30분(GAP 1800s) 내 무이벤트** = 1. **event-level**.
- **규모**: train **743,678** rows/60,000 users(2019-Nov, bounce **14.46%**), test **271,754**/30,000(2019-Dec, **18.43%**).
- **피처 8**: step·dt_prev_log·n_view_sf·n_cart_sf·n_purchase_sf·price_log·price_mean_sf_log·is_first(세션 누적).

### 6.5 보조 시퀀스
- **aux_nextcat**: L=10, 462 cat, full window **883,959** → 추린본 **143,915**(세션당 ≤4, 740,044 폐기/복원가능), 40,000 세션.
- **aux_fullsession**: raw **4,635,837** 행 → window **263,039**, 세션 **72,052**, 471 cat, L=10·H=5·C=8.

## 8. 라벨/클래스 검증 ④
- **churn 96.7% 근거**: 5개월 전체 유저의 다수가 **단발성(1~2일 활동 후 미복귀)** → recency가 커 대부분 이탈로 라벨됨. 그대로 학습하면 "거의 다 이탈" 자명 분류가 되어 변별력↓.
- **활성 코호트(recency≤7) 도입**: "최근 활동했음에도 7일 내 이탈"만 남겨 **base rate 82.17%**(train)·**83.3%**(test)로 낮춰 의미 있는 판별 문제로 전환. 모든 모델 평가는 이 코호트(n_test 109,736)에서 수행.
- **추천 클래스 정합**: 카테고리 outcome 커버리지 train 3.33% < test 9.77%(테스트 구간 라벨 보유율이 높음). majority top-1 5.54% 대비 모델 top-1 9~13%로 상회(§10).
- **카테고리 수가 산출물마다 다른 이유**(주의): **525**(원천 전체) → **389**(v4-1, MIN_COUNT≥10 희소필터) → **462**(aux_nextcat, 2019-Nov 표본) → **471**(fullsession) → **284**(일부 bayes의 학습셋 등장 클래스) → **516**(GRU vocab, +pad). 즉 **기간·표본·최소빈도·pad 포함 여부 차이**다.

## 9. 누수(leakage) 차단 검증 ⑤
- **시간외삽 분할**: train은 관측창(~2020-01-25)만, test는 **2020-02(OOT)**. 피처는 **관측창 데이터만** 집계, outcome 구간(라벨 산정 7일) 정보는 피처에 **미포함**.
- 검증: `techniques`에 `시간외삽분할(누수차단)` 명시 + 스케일러/인코더는 **train fit → test transform**(불변 규칙). cross-month 통계도 관측창 한정.

## 10. 하이퍼파라미터 선택 · 근거 · 결과 (핵심) ★

선택 절차: **모델군별 베이지안 탐색**으로 (전처리 옵션: scaler·log_counts·imbalance) × (모델 HP)를 동시 탐색 → **CV 주지표로 선택**, **OOT(2020-02)로 검증**. per-option 결과도 기록해 옵션 효과를 근거로 남김.

### 10.1 7일 이탈 — 모델별 best (지표: PR-AUC, base rate 0.8217) `*_v2_bayes.json`

| 모델 | scaler | log | imbalance | 핵심 HP | CV PR-AUC | OOT AUC | OOT PR-AUC | thr |
| --- | --- | --- | --- | --- | ---: | ---: | ---: | ---: |
| **XGBoost ★** | robust | F | classweight | n312·d5·lr0.036 | 0.9370 | **0.7905** | 0.9345 | 0.54 |
| **LightGBM** | standard | F | none | n399·leaves195·d3 | **0.9371** | 0.7905 | 0.9343 | 0.55 |
| **CatBoost** | none | F | none | iter303·d4·lr0.044 | 0.9371 | 0.7910 | 0.9347 | 0.52 |
| RandomForest | none | F | none | n164·d8·leaf33 | 0.9366 | 0.7900 | 0.9341 | 0.59 |
| LogReg | minmax | **T** | classweight | C=67.1 | 0.9359 | 0.7871 | 0.9328 | 0.59 |
| DecisionTree | none | F | classweight | d12·leaf35 | 0.9287 | 0.7754 | 0.9266 | 0.42 |

- **선택 근거**: 불균형(82%)이라 **PR-AUC 주지표**. 상위 3(XGB/LGBM/CatBoost)이 CV PR-AUC ~0.937 동률 + OOT AUC 0.79로 최상 → **best3 확정**. 트리계열은 scaler/log 거의 무관(none·F)했고, **LogReg만 minmax+log+classweight 필수**(선형 특성).
- **ML vs DL 전처리 분기 인사이트**(`bayes_5m.json`): ML(GBM)은 `minmax+log`(불균형 none/smote)에서 최적, **DL(LSTM)은 `norm=none`+`pos_weight=false`** 에서 최적 → "ML과 DL 최적 전처리가 **다름**"(`ml_vs_dl_prep_differ=true`)을 확인, 모델군별 전처리 동결 근거. (baseline LogReg CV AUC 0.7716 → 튜닝 후 0.7967)

## 11. 경계 명확화 ⑥
- **본 문서(전처리)** = 원시 피처/시퀀스/라벨 **생성**까지 + **모델별 전처리·HP 탐색 결과 기록**.
- **모델 파이프라인 단계** = scaler/log/imbalance(SMOTE·classweight)·인코딩을 **prep 번들(joblib)에 동결**해 적용(train fit→runtime transform). 즉 §10의 옵션들은 "탐색으로 정한 값"이며 실제 적용은 모델 번들에서 일어난다.
